terminal pre-flight for Codex

Build the prompt before Codex takes off.

Prompt Builder is a small terminal composer for the moment before execution. Use it to name the work, shape the prompt, add skills, reference files, inspect the command, and hand off only when the task is ready.

It sits between your shell and Codex. It does not replace Codex. It makes the launch cleaner.

why pre-flight matters

A prompt is not just text. It is a launch.

When the prompt is vague, the run starts vague. That means more correction, more context repair, and more time spent steering an agent that already left the runway.

Before

A half-formed prompt goes straight into execution.

During

The agent starts making assumptions while the real task is still forming.

After

The developer spends the session cleaning up intent.

what it does

Prompt Builder opens a focused terminal UI before handoff.

It supports a prompt field, optional conversation name, skill lookup, file lookup, handoff options, Codex pass-through flags, dry-run output, and structured launch JSON.

Compose

Write the real task before a tool that can edit files receives it.

Enrich

Add skill mentions and file references while the prompt is still forming.

Handoff

Submit to Codex or pass the final prompt as an argv element to a wrapper.

the loop

The pre-flight loop

Prompt Builder keeps the workflow boring in the best way: prepare the prompt before the command that can run tools receives it.

Open

Start empty, prefill text, or pipe content from another command.

Name

Give the work a short thread name when that helps the run stay legible.

Compose

Use Enter to submit and Shift+Enter for new lines.

Add context

Use $ for skills and @ for files.

Choose route

For wrappers, toggle options like fork, compact, or custom args.

Inspect or submit

Print the prompt, command, launch JSON, dry-run, or submit.

user stories

Where it earns its keep

Prompt Builder is for the moments where a little structure before execution prevents a messy session afterward.

Turn "fix this" into an executable task.

A developer starts with a rough idea, but does not want Codex acting on it yet. Prompt Builder gives them a staging area to name the thread, write the real task, add verification instructions, and only then send it.

Beforefix focus bug
AfterInspect the focus flow, reproduce, patch, and verify.

Route the next move through the right wrapper.

A user working from a focused Codex pane can compose the next prompt and hand it through a wrapper with fork and compact options visible. Prompt Builder does not own wrapper orchestration; it forwards selected option tokens and the final prompt.

Beforecontinue from last, but compact first
Afterhandoff args + enabled toggles + final prompt

Add the right skill while the prompt is still being written.

Instead of remembering exact skill names or typing them cold, the user opens skill lookup with $, filters the list, and inserts the selected skill into the prompt.

Beforeuse fusion somehow
AfterStart with $fusion, inspect the failing path, patch, then verify.

Reference files before the handoff.

The user types an @ reference, searches the project file list, accepts the right path, and submits a prompt that names the files Codex should inspect first.

Beforeinspect @ma
Afterinspect src/main.rs

Inspect the launch before it runs.

For shell, cmux, or wrapper workflows, Prompt Builder can print the composed prompt, render the command, dry-run submission, or emit structured launch JSON so other tools do not parse shell-quoted text.

Beforepipe text into a tool and hope
Afterprint prompt / print command / launch JSON / dry-run

CLI examples

Practical commands for the workflows this page describes.

Open and compose
prompt-builder
prompt-builder "fix the focus in the main window"
Submit from stdin
printf 'fix the focus in the main window\n' | \
  prompt-builder --submit
Print for pipelines
printf 'explain this\n' | \
  prompt-builder --stdin --print-prompt
Inspect before launch
prompt-builder --dry-run --print-command "hello"
prompt-builder --print-launch-json "hello"
Handoff mode
prompt-builder --handoff-command xfc
prompt-builder --handoff-command x \
  --handoff-arg fork \
  --handoff-arg=--last
Fork and compact toggles
prompt-builder \
  --handoff-command xfc \
  --fork-from last \
  --compact
Codex pass-through
prompt-builder \
  -C ~/dev/codex \
  --profile fixit \
  --model gpt-5.5 \
  -c 'instructions="Be concise."'
Instruction shorthands
prompt-builder --instructions "Base instructions"
prompt-builder --developer-instructions \
  "Inspect source first."
what it is not

Boundaries and safety

Prompt Builder should be explained plainly. Its power comes from being small.

Not a Codex replacement.
It composes and routes the prompt before Codex or a handoff command receives it.
Not the wrapper orchestrator.
Fork, compact, and app-server behavior belong to the wrapper. Prompt Builder forwards selected argv tokens.
Not shell eval.
Handoff mode passes the prompt as an argument, not by evaluating shell text.
Default Codex launch is powerful.
The default launcher uses --dangerously-bypass-approvals-and-sandbox. Use trusted directories and inspect before running.
Blank prompts do not launch.
Empty submissions are rejected before handoff.
Small by design

This page does not imply a hosted dashboard, account system, cloud sync, prompt marketplace, analytics layer, or broad roadmap.
